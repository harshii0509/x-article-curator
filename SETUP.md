## Local Setup Guide

This document explains how to run the Nightstand web app and browser extension on your local machine.

### 1. Prerequisites

- **Node.js**: v18 or newer
- **npm**: comes with Node (or use pnpm/yarn if you prefer, but examples use npm)
- **SQLite**: optional but useful if you want to inspect the local database (`sqlite.db`)
- **Google Cloud account**: only needed if you want to create your own OAuth credentials instead of using shared ones

The repository has two main parts:

- Web app: `web/` (Next.js)
- Chrome extension: `extension/`

---

### 2. Web app setup

From the repo root:

1. **Install dependencies**

   ```bash
   cd web
   npm install
   ```

2. **Create your local env file**

   Copy the example file and fill in the values:

   ```bash
   cd web
   cp .env.example .env.local
   ```

   Required variables in `.env.local`:

   - `API_SECRET_KEY`: Shared secret used for the global API key auth path. Any long random string is fine for local dev.
   - `GOOGLE_CLIENT_ID`: Google OAuth client ID (backend verifier).
   - `NEXT_PUBLIC_GOOGLE_CLIENT_ID`: Same Google OAuth client ID, but exposed to the frontend (must match `GOOGLE_CLIENT_ID`).
   - `ALLOWED_ORIGINS`: Comma-separated list of allowed CORS origins for the API.
     - For local dev, usually: `http://localhost:3000`

3. **Create / sync the database**

   The app uses a local SQLite database in `web/sqlite.db`, managed via Drizzle.

   ```bash
   cd web
   npm run db:push
   ```

   This will create or migrate `sqlite.db` to match the current schema.

4. **Run the dev server**

   ```bash
   cd web
   npm run dev
   ```

   The app will be available at:

   - `http://localhost:3000`

5. **(Optional) Browse the DB with Drizzle Studio**

   ```bash
   cd web
   npm run db:studio
   ```

   This opens a UI to inspect tables and data in `sqlite.db`.

---

### 3. Chrome extension setup

The extension lets you save the current page to your Nightstand reading list.

1. **Load the extension in Chrome**

   - Go to `chrome://extensions`
   - Enable **Developer mode**
   - Click **Load unpacked**
   - Select the `extension/` folder from this repo

2. **Configure the API URL and token (options page)**

   - In the extensions list, click **Details** on the Nightstand extension
   - Click **Extension options**
   - Set:
     - **API URL**: `http://localhost:3000` (for local dev)
     - **Personal API token**: your API token from the web app (after logging in)

3. **Google OAuth client ID**

   The extension hardcodes a Google OAuth client ID in:

   - `extension/popup/popup.js`

   That value must match the `GOOGLE_CLIENT_ID` / `NEXT_PUBLIC_GOOGLE_CLIENT_ID` you configure in `web/.env.local`. If you ever rotate the client ID, update it in both places.

---

### 4. Google OAuth setup (if you need your own credentials)

If shared Google OAuth credentials are not provided, you can create your own:

1. Go to **Google Cloud Console** → create/select a project.
2. Configure the **OAuth consent screen**.
3. Create **OAuth 2.0 Client ID** (type: **Web application**).
4. Add **Authorized JavaScript origins**:
   - `http://localhost:3000` (for local dev web app)
5. Add **Authorized redirect URIs**:
   - As required by your frontend login flow.
6. Copy the client ID and set it in:
   - `GOOGLE_CLIENT_ID` in `web/.env.local`
   - `NEXT_PUBLIC_GOOGLE_CLIENT_ID` in `web/.env.local`
   - `GOOGLE_CLIENT_ID` constant in `extension/popup/popup.js`

For the extension’s `chrome.identity` flow, you may also need to add the redirect URI generated by:

```js
chrome.identity.getRedirectURL("oauth2")
```

to the Google Cloud console as an authorized redirect URI.

---

### 5. Typical local dev workflow

1. Start the web app:

   ```bash
   cd web
   npm run dev
   ```

2. Load the extension from `extension/` in Chrome (Developer mode).
3. Log in via Google on the web app and copy your API token if needed.
4. Configure the extension options (API URL + token).
5. Use the extension’s popup to save links while browsing.
